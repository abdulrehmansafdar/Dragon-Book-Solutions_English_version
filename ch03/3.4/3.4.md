### 3.4.4 ！

By induction on s, prove that the algorithm of Figure 3-19 correctly calculates the failure function.

**Figure 3-19: Calculating the failure function of keywords b_1b_2…b_n**

01  t = 0;
02  f(1) = 0;
03  for (s = 1; s < n; s ++){
04      while( t > 0 && b_s+1 != b_t+1) t = f(t);
05      if(b_s+1 == b_t+1){
06          t = t + 1;
07          f(s + 1) = t;
08      }else{
09          f(s + 1) = 0;
10      }
11  }

#### prove

1. It is known that f(1) = 0
2. In the first for loop, calculate the value of f(2). When the 5th line of code b_2 == b_1 is established, the code goes to line 7 and f(2) = 1 is obtained. If it is not established, the code goes to line 9 and f(2) = 0 is obtained.Obviously, this loop correctly calculated f(2).
3. Assume that f(i) is calculated correctly when entering the loop for the i-1th time, and f(i) = t (regardless of whether t is greater than 0 or equal to 0)
4. Then when entering the loop for the first time, two situations will be considered:
1.t==0

This situation is relatively simple, starting directly from line 5, when b_i+1 == b_1, f(i+1) = 1, otherwise f(i+1) = 0
2. t > 0
The while loop will continue to reduce the t value, trying to find the maximum possible t value that makes b_i+1 == b_t+1 true. If it is found, it will enter line 5 for execution and get f(i+1) = t+1; or it will not find it until t == 0, then it will jump out of the loop and enter line 5 for execution. The process is similar to the previous situation.
